# Melampo Clinical Memory Dataset Card

## Status

Template and governance baseline. No patient dataset is bundled with this repository.

## Dataset purpose

Melampo's semantic memory is intended to store governed clinical and research objects:

- de-identified case traces;
- clinical reports;
- imaging study metadata and model-derived findings;
- literature and guideline chunks;
- epidemiological factors;
- ontology references;
- synthetic dream/replay candidates.

## Data classes

Target Weaviate object classes:

- Symptom
- Pathology
- ClinicalCase
- ImagingStudy
- ImagingFinding
- ClinicalDocument
- EpidemiologicalFactor

## Required provenance

Every record must include:

- source identifier;
- ingestion timestamp;
- license or access class;
- de-identification status;
- original modality;
- parser/model version where applicable;
- learning status (`candidate`, `promoted`, `rejected`, `needs_review`, `retired`);
- audit trace if generated by Melampo.

## Synthetic data policy

Dream-generated and counterfactual records must default to `candidate`.

They must not be promoted unless:

1. rational-control validation passes;
2. contradictions are documented;
3. provenance marks the record as synthetic;
4. a reviewer approves promotion;
5. clinical use remains blocked until formal validation.

## Bias and safety risks

Potential risks:

- demographic imbalance;
- site/scanner bias;
- referral bias;
- prevalence drift;
- rare disease underrepresentation;
- hallucinated literature chunks;
- synthetic traces contaminating real evidence;
- ontology code mismatch.

## Required evaluation slices

At minimum, evaluation should slice by:

- modality;
- age band;
- sex/gender where available and lawful;
- geography/site;
- scanner/protocol;
- pathology family;
- disease prevalence band;
- source type: real, literature, guideline, synthetic;
- learning status.

## Release and access

The repository contains only schema/contracts and no clinical data. Any future dataset release must have its own dataset card, license, ethics review, de-identification report and access policy.
